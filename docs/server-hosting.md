## 現状の課題（Hamachi）

現在サーバーは **Hamachi VPN** を使って接続しています。Hamachi は手軽ですが、以下の課題があります。

| 課題 | 説明 |
|------|------|
| **全員にインストール必須** | プレイヤー全員が Hamachi をインストール・ログイン・ネットワーク参加する必要がある |
| **無料版の制限** | 1 ネットワークあたり最大 5 人まで（有料版が必要になる場合あり） |
| **接続の不安定さ** | VPN 経由のためラグが増える。Hamachi 自体の障害で接続不能になることも |
| **追加のレイテンシ** | VPN トンネルを経由するため +20〜50ms 程度の遅延が生じる |
| **トラブルシューティングの困難さ** | Hamachi 側の問題か Minecraft 側の問題かの切り分けが難しい |
| **プレイヤー体験の悪化** | 「Hamachi って何？」から始まるセットアップが参入障壁になる |

**目標**: プレイヤーは **Minecraft のサーバーアドレスを入力するだけ** で接続できる状態にする。

## 自宅サーバー方式

自宅の PC やサーバーで Minecraft サーバーを動かし、外部に公開する方式です。

**メリット**: ランニングコストが低い（電気代のみ）、自由度が高い
**デメリット**: ネットワーク設定が必要、ISP の制約を受ける

---

### A. ポートフォワーディング（直接公開）

最もシンプルで高速な方式。ルーターの設定で TCP ポート 25565 を Minecraft サーバーの内部 IP に転送します。

**条件**:
- グローバル IPv4 アドレスを持っていること（CGNAT でないこと）
- ルーターの管理画面にアクセスできること

**手順**:
1. ルーター管理画面（通常 `http://192.168.11.1` や `http://192.168.1.1`）にアクセス
2. 「ポートフォワーディング」「ポート転送」「仮想サーバー」等の設定を探す
3. 以下を設定：
   - プロトコル: TCP
   - 外部ポート: 25565
   - 内部 IP: サーバー PC のローカル IP（例: `192.168.11.23`）
   - 内部ポート: 25565
4. プレイヤーはグローバル IP でサーバーに接続

**メリット**: 遅延ゼロ（最速）、追加ソフト不要
**デメリット**: ルーターへのアクセスが必要、セキュリティ上の注意が必要

> ⚠️ **セキュリティ注意**: ポートを公開するとインターネットから直接アクセス可能になります。
> Minecraft サーバーには whitelist を有効にし、不要なポートは開けないでください。

**ポートフォワーディングができない場合**:
- ISP が CGNAT（キャリアグレード NAT）を使用している
- マンション共用のルーターで設定権限がない
- ISP がポート開放を制限している

→ 以下の B, C の方式を検討してください。

### B. IPv6 直接接続

多くの ISP では IPv6 グローバルアドレスが割り当てられています。IPv4 でポートフォワーディングができなくても、IPv6 なら直接接続できる場合があります。

**条件**:
- サーバー側・クライアント側の両方が IPv6 に対応していること
- **Minecraft Java Edition** であること（Bedrock Edition は IPv6 非対応）

**確認方法（サーバー側）**:

| OS | コマンド |
|----|----------|
| macOS | `ifconfig en0 \| grep 'inet6' \| grep -v 'fe80'` |
| Linux | `ip -6 addr \| grep 'inet6' \| grep -v 'fe80'` |
| Windows (PowerShell) | `ipconfig \| Select-String 'IPv6'` |

どの OS でも以下で IPv6 が使えるか確認できます：
```bash
curl -s -6 https://ipv6.icanhazip.com
```

`2001:` や `2400:` で始まるアドレスが表示されれば IPv6 が使えます。

**Minecraft サーバーの設定**:
`server.properties` で以下を確認：
```properties
server-ip=
# server-ip を空のままにすれば IPv4/IPv6 の両方でリッスンします
```

**クライアント側の接続**:
```
サーバーアドレス: [2401:xxxx:xxxx:xxxx::1]
# IPv6 アドレスは [] で囲む
```

**メリット**: ポートフォワーディング不要、NAT を経由しないので低遅延
**デメリット**: クライアント側も IPv6 必須（全世界で約 45〜50% の普及率）、アドレスが長くて覚えにくい

> ⚠️ **セキュリティ注意**: IPv6 は NAT を経由しないため、サーバー PC が直接インターネットに露出します。
> OS のファイアウォールを有効にし、必要なポート（25565）のみ許可してください。
>
> | OS | ファイアウォール確認コマンド |
> |----|------------------------|
> | macOS | `sudo /usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate` |
> | Linux | `sudo ufw status` または `sudo firewall-cmd --list-all` |
> | Windows (PowerShell) | `Get-NetFirewallProfile \| Format-Table Name,Enabled` |

### C. playit.gg（トンネルサービス）

[playit.gg](https://playit.gg) は Minecraft サーバーを外部に公開するためのトンネルサービスです。ポートフォワーディングや IPv6 が使えない場合の代替手段として最適です。

**仕組み**:
```
プレイヤー ──→ playit.gg サーバー ──→ 自宅の Minecraft サーバー
              （中継サーバー）         （トンネルで接続）
```

**特徴**:
| 項目 | 内容 |
|------|------|
| 料金 | **無料プラン**あり（個人利用には十分） |
| セットアップ | playit エージェントをサーバーに常駐させるだけ |
| ドメイン | `xxxxx.joinmc.link` のようなアドレスが自動付与 |
| 対応 | Java Edition / Bedrock Edition 両対応 |

**セットアップ手順**:
1. [playit.gg](https://playit.gg) でアカウント作成
2. playit エージェントをダウンロード・インストール

   | OS | インストール方法 |
   |----|----------------|
   | macOS | `brew install playit` |
   | Windows | [playit.gg/download](https://playit.gg/download) からインストーラーをダウンロード |
   | Linux | `curl -SsL https://playit-cloud.github.io/ppa/key.gpg \| sudo apt-key add -` → [playit.gg/download](https://playit.gg/download) 参照 |

3. エージェントを起動して認証
   ```
   playit setup
   ```
4. ダッシュボードでトンネルを作成（Minecraft Java, ポート 25565）
5. 発行されたアドレス（例: `xxxxx.joinmc.link`）をプレイヤーに共有

**メリット**: ISP の制約を完全に回避、セットアップが簡単、プレイヤーは普通にアドレスで接続
**デメリット**: playit.gg サーバーを経由するため **+10〜80ms のレイテンシ**が追加される

> 💡 **Hamachi との比較**: playit.gg はプレイヤー側に追加ソフトのインストールが不要です。
> これが Hamachi から移行する最大のメリットです。

### D. DDNS（ダイナミック DNS）

ほとんどの家庭用インターネット回線は **動的 IP**（IP アドレスが定期的に変わる）です。DDNS を使うと、固定のドメイン名で常にサーバーにアクセスできるようになります。

**仕組み**:
```
プレイヤー  →  minecraft.dynv6.net  →  DNS 解決  →  現在の IP アドレス  →  サーバー
                 （固定ドメイン）                    （自動更新）
```

**主要な無料 DDNS サービス**:

| サービス | ドメイン例 | IPv6 対応 | 更新方法 |
|---------|-----------|----------|---------|
| [dynv6](https://dynv6.com) | `yourname.dynv6.net` | ✅ | HTTP API / スクリプト |
| [DuckDNS](https://www.duckdns.org) | `yourname.duckdns.org` | ✅ | HTTP API / cron |
| [No-IP](https://www.noip.com) | `yourname.ddns.net` | ❌（有料） | クライアント常駐 |

**推奨: dynv6**（IPv6 完全対応、完全無料、API がシンプル）

**自動更新スクリプトの例（dynv6）**:

<details>
<summary>macOS / Linux (bash + cron)</summary>

```bash
#!/bin/bash
# 5分ごとに cron で実行
TOKEN="your-dynv6-token"
HOSTNAME="yourname.dynv6.net"

# IPv4 更新
IPV4=$(curl -s https://ipv4.icanhazip.com)
curl -s "https://dynv6.com/api/update?hostname=${HOSTNAME}&ipv4=${IPV4}&token=${TOKEN}"

# IPv6 更新
IPV6=$(curl -s https://ipv6.icanhazip.com)
curl -s "https://dynv6.com/api/update?hostname=${HOSTNAME}&ipv6=${IPV6}&token=${TOKEN}"
```

</details>

<details>
<summary>Windows (PowerShell + タスクスケジューラー)</summary>

```powershell
# update-ddns.ps1 として保存し、タスクスケジューラーで5分ごとに実行
$Token = "your-dynv6-token"
$Hostname = "yourname.dynv6.net"

$IPv4 = (Invoke-RestMethod https://ipv4.icanhazip.com).Trim()
Invoke-RestMethod "https://dynv6.com/api/update?hostname=$Hostname&ipv4=$IPv4&token=$Token"

$IPv6 = (Invoke-RestMethod https://ipv6.icanhazip.com).Trim()
Invoke-RestMethod "https://dynv6.com/api/update?hostname=$Hostname&ipv6=$IPv6&token=$Token"
```

</details>

**信頼性**:
- DDNS 自体は 99% 以上の稼働率
- IP 変更時に DNS キャッシュが残るため、**10〜35 分程度の接続不能時間**が生じる可能性がある
- IP 変更の頻度は ISP によるが、一般的には数週間〜数ヶ月に 1 回程度

> 💡 **DDNS は単体では使えません**: ポートフォワーディング（A）または IPv6 直接接続（B）と組み合わせて使います。
> playit.gg（C）を使う場合は、playit が固定ドメインを提供するため DDNS は不要です。

### E. 推奨構成: IPv6 + playit.gg + DDNS

自宅サーバーの推奨構成は、複数の方式を組み合わせたハイブリッド構成です。

```
                    ┌─────────────────────────────────┐
                    │        プレイヤー                  │
                    └──────────┬──────────────────────┘
                               │
                    ┌──────────▼──────────┐
                    │   IPv6 対応？        │
                    └──┬──────────────┬───┘
                  Yes  │              │  No
                       ▼              ▼
              ┌────────────┐  ┌──────────────┐
              │ IPv6 直接   │  │ playit.gg    │
              │ (低遅延)    │  │ (IPv4 経由)  │
              └──────┬─────┘  └──────┬───────┘
                     │               │
                     ▼               ▼
              ┌──────────────────────────┐
              │     Minecraft サーバー     │
              │     （自宅 Mac mini）       │
              └──────────────────────────┘
```

**プレイヤーへの案内**:
1. まず IPv6 アドレス（DDNS ドメイン）で接続を試す → 繋がれば最速
2. 繋がらない場合は playit.gg のアドレスで接続 → 誰でも繋がる

**サーバー管理者が準備するもの**:
- [ ] DDNS の設定（dynv6 等）で IPv6 アドレスを固定ドメインに紐付け
- [ ] playit.gg エージェントの常駐起動
- [ ] OS のファイアウォールで 25565 ポートを許可
- [ ] `server.properties` で `server-ip=` を空に設定

## レンタルサーバー方式（exaroton）

自宅にサーバーを置きたくない、またはネットワーク環境の制約が大きい場合は、レンタルサーバーが選択肢になります。

[exaroton](https://exaroton.com) は Minecraft に特化した従量課金型のホスティングサービスです。

**特徴**:
| 項目 | 内容 |
|------|------|
| 課金単位 | **サーバー起動時間**に対して課金（停止中は無料※） |
| 料金目安 | 4GB RAM で約 $0.04/時間（≒ ¥6/時間） |
| 自動停止 | プレイヤーがいなくなると自動停止（設定可能） |
| 自動起動 | プレイヤーが接続すると自動起動 |
| Mod 対応 | Forge / Fabric / Paper 等すべて対応 |
| ストレージ | 停止中も ≒ ¥19/月 のストレージ費用が発生 |

**月額コスト目安**（4GB RAM プラン）:

| プレイ時間 | 月額 |
|-----------|------|
| 0時間（誰もプレイしない） | 約 ¥19（ストレージのみ） |
| 週5時間（月20時間） | 約 ¥120 + ¥19 = **約 ¥139** |
| 週15時間（月60時間） | 約 ¥360 + ¥19 = **約 ¥379** |
| 毎日5時間（月150時間） | 約 ¥900 + ¥19 = **約 ¥919** |

**メリット**: 使った分だけ課金、サーバー管理不要、プレイヤーからの接続性が良い
**デメリット**: ヘビーに使うと自宅サーバーより高い、Mod のアップロードが手間

> 💡 **自動起動機能**: プレイヤーがサーバーアドレスに接続しようとすると自動でサーバーが起動します。
> 起動完了まで 30〜60 秒程度かかりますが、待てば自動接続されます。

## ISP 環境による制約と確認方法

サーバー公開方式を選ぶ前に、自分のネットワーク環境を確認しましょう。

### 1. グローバル IP の確認

```bash
curl -s https://ipinfo.io
```

結果の `org` フィールドで ISP がわかります。`ip` フィールドが自分のグローバル IP です。

### 2. CGNAT の判定

```bash
# ルーターの WAN 側 IP を確認
# ルーター管理画面で「WAN IP」「インターネット IP」等を確認

# グローバル IP と一致すれば CGNAT ではない
curl -s https://ipv4.icanhazip.com
```

- **一致** → グローバル IP（ポートフォワーディング可能）
- **不一致** → CGNAT の可能性が高い → playit.gg を使う

### 3. IPv6 の確認

```bash
# IPv6 アドレスの確認
curl -s -6 https://ipv6.icanhazip.com

# 応答があれば IPv6 が使える
```

### 4. 判断フローチャート

```
ルーターの管理画面にアクセスできる？
  │
  ├─ Yes → ポートフォワーディングを設定できる？
  │          ├─ Yes → 方式 A（ポートフォワーディング）+ DDNS ✅ 最速
  │          └─ No  → ISP が制限している可能性
  │
  └─ No → マンション共用ルーター等
           │
           IPv6 が使えるか確認
           ├─ Yes → 方式 B（IPv6 直接）+ playit.gg 併用 ✅ 推奨
           └─ No  → 方式 C（playit.gg のみ）✅ 確実
```

## レイテンシについて

Minecraft のプレイ体験に最も影響するのはレイテンシ（遅延）です。

### 接続方式によるレイテンシへの影響

| 方式 | 追加レイテンシ | 体感 |
|------|-------------|------|
| ポートフォワーディング / IPv6 直接 | 0ms | 接続方式による遅延なし |
| playit.gg | +10〜80ms | 通常のプレイには問題ない |
| Hamachi | +20〜50ms | たまにラグを感じる |

### サーバー所在地による影響（物理的距離）

| プレイヤーの地域 | サーバーが日本 | サーバーが米国 |
|----------------|-------------|-------------|
| 日本 | 5〜20ms | 100〜180ms |
| 東南アジア | 30〜80ms | 150〜250ms |
| 北米 | 100〜180ms | 10〜40ms |
| 欧州 | 200〜300ms | 80〜120ms |

**日本と海外の両方にプレイヤーがいる場合**、物理法則によりどちらかが遠くなるのは避けられません。自宅サーバー（日本）なら日本のプレイヤーが有利になります。

> 💡 **目安**: 100ms 以下なら快適にプレイ可能。200ms を超えるとラグが目立ち始めます。

## 移行手順（Hamachi → 新方式）

Hamachi から移行する場合、プレイヤーに影響を与えないよう段階的に進めます。

### Phase 1: 並行運用（推奨）

1. 新しい接続方式（playit.gg / IPv6 / ポートフォワーディング）をセットアップ
2. 新しいサーバーアドレスをプレイヤーに共有
3. Hamachi は残したまま、新アドレスでの接続を促す
4. 全員が新アドレスで接続できることを確認

### Phase 2: Hamachi の撤去

1. README のサーバー接続手順を更新（Hamachi の記述を削除）
2. Hamachi のネットワークを削除
3. プレイヤーに Hamachi のアンインストールを案内

### サーバーアドレスの案内テンプレート

```
🎮 サーバー接続方法が変わりました！

Hamachi は不要になりました。以下のアドレスで直接接続できます。

【推奨】IPv6 対応の方:
  サーバーアドレス: minecraft.dynv6.net

【IPv6 非対応の方】:
  サーバーアドレス: xxxxx.joinmc.link

※どちらで繋がるかわからない場合は、まず上のアドレスを試して、
  ダメなら下のアドレスで接続してください。
```

## 参考: SleepingServerStarter（自宅サーバー向け）

自宅サーバーで「プレイヤーがいない間はサーバーを停止したい」場合、[SleepingServerStarter](https://github.com/vincss/mcsleepingserverstarter) が使えます。

**仕組み**: 25565 ポートで待ち受けるプロキシとして動作し、接続が来たら Minecraft サーバーを起動、全員ログアウトしたら停止します。

**メリット**: 電力・メモリの節約、exaroton の自動起動と同等の機能を自宅で実現
**注意**: 起動完了まで 30〜60 秒待つ必要がある
